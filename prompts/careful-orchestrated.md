# Careful Orchestrated Execution Profile

Use `careful` for broad or bounded work that benefits from an orchestrated topology and strong guardrails.

Recommended flow: scout/context pass, concise plan, bounded execution, required verification, code review, and evidence-based return. Use review/fix-loop guardrails for high-risk or larger changes.

Escalate back to the parent if the task needs decisions outside the handoff or scope expansion beyond the stated guardrails.
